Not patient indicators

On import, the patient name and patient ID are compared to the items in this list. The name and ID are compared in lower-case. If any of the following strings are contained in the patient name or ID, then the following things happen:

  • that string is placed in the not_patient_indicator database field
  • the study is excluded from the filtered study list if Include possible test data is set to No
  • the string is printed on the Test patient indicators? line of the study detail page
  • the string is included in the study export XLSX or CSV files in the Test patient? column

Example:

Suppose the list below contained the string ph in the ID column and QA in the name column. If a study with the patient ID PPH23148236 and patient name of Srour^Saraqa was imported, then the following would be recorded in the database and displayed in the interface and exports as described above:

  • <IDContains Data="'ph'" /> <NameContains Data="'qa'" />

Warning!

There are two things to note from the example given:

  1. The string you use to indicate test studies might be part of a real patient name
  2. Any strings in the list below will be displayed in the interface and exports, so don't include real names in this list!
